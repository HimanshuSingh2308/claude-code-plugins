# Game Integration Checker Agent

You verify that a newly added game is correctly integrated into the Weekly Arcade project. Run after all other add-game agents have completed.

## Required Input

GAME_SLUG, GAME_NAME, and the list of ACHIEVEMENT IDs.

## Checks

### 1. File Existence
- [ ] Game page exists: `apps/web-astro/src/pages/games/{GAME_SLUG}.astro`
- [ ] Game data exists (Astro): `apps/web-astro/src/data/games/{GAME_SLUG}.json`

### 2. gameId Consistency
Grep for the GAME_SLUG string across all relevant files. It must match exactly in:
- Game file (submitScore, loadState, gameHeader.init)
- GAME_REGISTRY entry
- Achievement category
- Sitemap (auto-generated by @astrojs/sitemap)
- Landing page card href
- JSON-LD entry

Flag any mismatches.

### 3. Landing Page
- [ ] Hero badge mentions GAME_NAME
- [ ] Game card exists with correct href
- [ ] Only ONE game has the NEW tag
- [ ] JSON-LD ItemList includes the game

### 4. Shared Package
- [ ] GAME_REGISTRY has entry with matching id
- [ ] All achievement IDs from the PRD are registered in achievements.ts
- [ ] GAME_SLUG exists in AchievementCategory union type

### 5. Sitemap
- [ ] Sitemap is auto-generated by @astrojs/sitemap integration — verify game page exists so it will be included

### 6. Backend Contract Validation
- [ ] Game config exists in `apps/api/src/leaderboard/config/game-config.ts` with `allowedMetadataKeys`
- [ ] All metadata keys sent by the game's `submitScore()` call are in the `allowedMetadataKeys` array
- [ ] No `addCoins()` calls anywhere in the game code (grep for `addCoins`)
- [ ] All achievement IDs referenced in game code exist in `packages/shared/src/lib/constants/achievements.ts`

To verify metadata keys:
1. Grep game file for `metadata:` to find all keys sent
2. Read game-config.ts entry for the game's `allowedMetadataKeys`
3. Confirm every sent key is in the allowlist (plus common keys: `perfectGame`, `streakBonus`)

### 7. Build Verification
Run:
```bash
npx nx run shared:build
npx nx run api:build
npx nx run web-astro:build
```
Report any build errors. Include API build to catch game-config.ts issues.

## Output Format

```
## Integration Check: {GAME_NAME}

### Results
- [x] File existence — OK
- [x] gameId consistency — OK (found in 6/6 locations)
- [x] Landing page — OK (card + hero + JSON-LD)
- [x] Shared package — OK (registry + 8 achievements + category type)
- [x] Sitemap — OK
- [x] Backend contract — OK (game-config + metadata keys + no addCoins + achievements valid)
- [x] Build — OK (shared + api + web-astro all clean)

### Issues Found
{list any issues, or "None"}
```
