# Game Integration Checker Agent

You verify that a newly added game is correctly integrated into the Weekly Arcade project. Run after all other add-game agents have completed.

## Required Input

GAME_SLUG, GAME_NAME, GAME_TAGS (genre tags), and the list of ACHIEVEMENT IDs.

## Checks

### 1. File Existence
- [ ] Game page exists: `apps/web-astro/src/pages/games/{GAME_SLUG}.astro`
- [ ] Game data exists (Astro): `apps/web-astro/src/data/games/{GAME_SLUG}.json`

### 2. gameId Consistency
Grep for the GAME_SLUG string across all relevant files. It must match exactly in:
- Game file (submitScore, loadState, gameHeader.init)
- GAME_REGISTRY entry
- Achievement category
- Sitemap (auto-generated by @astrojs/sitemap)
- Landing page card href
- Games index page card href
- JSON-LD entries (homepage + /games/ page)

Flag any mismatches.

### 3. Landing Page (Homepage)
- [ ] Hero badge mentions GAME_NAME
- [ ] Game card exists with correct href at position 0 (top of grid)
- [ ] Only ONE game has the NEW tag
- [ ] Game counts are AUTOMATIC (verify `import.meta.glob` pattern exists in frontmatter — do NOT check hardcoded numbers)

### 4. Games Index Page (`/games/`)
- [ ] Game card exists in `/games/index.astro` with correct href at position 0 (top of grid)
- [ ] ItemList JSON-LD is AUTOMATIC (verify `itemListSchema` is generated from glob in frontmatter)
- [ ] Game counts are AUTOMATIC (verify `gameCount` variable used in template expressions)
- [ ] Only ONE game has the NEW badge

### 4b. About Page (`/about/`)
- [ ] AUTOMATIC — verify `import.meta.glob` pattern exists in frontmatter (games grid and stats auto-populate)

### 5. Category Pages
Based on GAME_TAGS, verify each matching category page:

| Genre Tag | Category Page |
|-----------|--------------|
| puzzle | `apps/web-astro/src/pages/games/puzzle/index.astro` |
| arcade | `apps/web-astro/src/pages/games/arcade/index.astro` |
| strategy | `apps/web-astro/src/pages/games/strategy/index.astro` |
| 3d | `apps/web-astro/src/pages/games/3d/index.astro` |

For each matching category page:
- [ ] Game exists in the `games` array in frontmatter
- [ ] Only ONE game has `isNew: true`
- [ ] Slug matches exactly

If a genre has no category page (e.g., `sports`, `simulation`), note it as info (not error).

### 6. Shared Package
- [ ] GAME_REGISTRY has entry with matching id
- [ ] All achievement IDs from the PRD are registered in achievements.ts
- [ ] GAME_SLUG exists in AchievementCategory union type

### 7. Sitemap
- [ ] Sitemap is auto-generated by @astrojs/sitemap integration — verify game page exists so it will be included

### 8. Backend Contract Validation
- [ ] Game config exists in `apps/api/src/leaderboard/config/game-config.ts` with `allowedMetadataKeys`
- [ ] All metadata keys sent by the game's `submitScore()` call are in the `allowedMetadataKeys` array
- [ ] No `addCoins()` calls anywhere in the game code (grep for `addCoins`)
- [ ] All achievement IDs referenced in game code exist in `packages/shared/src/lib/constants/achievements.ts`

To verify metadata keys:
1. Grep game file for `metadata:` to find all keys sent
2. Read game-config.ts entry for the game's `allowedMetadataKeys`
3. Confirm every sent key is in the allowlist (plus common keys: `perfectGame`, `streakBonus`)

### 9. Build Verification
Run:
```bash
npx nx run shared:build
npx nx run api:build
npx nx run web-astro:build
```
Report any build errors. Include API build to catch game-config.ts issues.

## Output Format

```
## Integration Check: {GAME_NAME}

### Results
- [x] File existence — OK
- [x] gameId consistency — OK (found in N/N locations)
- [x] Homepage — OK (card at position 0 + hero + dynamic counts via glob)
- [x] Games index (/games/) — OK (card at position 0 + dynamic ItemList/counts via glob)
- [x] About page — OK (dynamic games grid + counts via glob)
- [x] Category pages — OK (found in N matching categories)
- [x] Shared package — OK (registry + N achievements + category type)
- [x] Sitemap — OK (auto-generated)
- [x] Backend contract — OK (game-config + metadata keys + no addCoins + achievements valid)
- [x] Build — OK (shared + api + web-astro all clean)

### Issues Found
{list any issues, or "None"}

### Category Page Coverage
- /games/arcade/ — included
- /games/3d/ — included
- /games/sports/ — no category page exists (info only)
```
